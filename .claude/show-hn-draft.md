```
Hi HN, I'm [name]. I built Tulmek because I was drowning in scattered
interview prep content — Reddit threads, Medium posts, dev.to articles,
LeetCode discussions — with no way to know what was actually worth reading.

Tulmek aggregates 750+ articles from 8 sources and ranks them with TCRA,
a multi-signal algorithm that weighs content richness, freshness decay,
source credibility, engagement velocity, trending detection, category
diversity, and personalization. Deterministic, transparent scoring — not
"AI-recommended" magic.

The whole thing is offline-first. No server, no auth, no database, no
tracking. Content ships as static JSON, search is client-side (Orama),
bookmarks saved in localStorage. You open it, it works.

Key features:
- 39 company intelligence pages with hiring signals and interview profiles
- Weekly Market Pulse auto-generated from aggregated data
- RSS feed, shareable OG images, keyboard-first navigation
- Cross-platform: web + desktop (Tauri v2) + mobile (Expo SDK 55)

Stack: Next.js 16, Tailwind v4, Turborepo monorepo, clean architecture
(ports/adapters), zero-dep shared core package. MIT licensed.

GitHub: https://github.com/coast-guide-sahil/tulmek

What I'm curious about: how do you organize your interview prep? Do you
bookmark articles and forget them, or do you have a system?
```
